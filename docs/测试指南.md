# 完整测试指南 - Mechanical Engineering AI System

## 📋 前置条件

确保系统已启动：
```bash
# 方式1: Docker启动
docker-compose up -d

# 方式2: 本地启动
./startup.sh          # macOS/Linux
./startup.ps1         # Windows
```

## 🚀 快速开始

### 1️⃣ 安装测试依赖

```bash
pip install pytest httpx pytest-asyncio
```

### 2️⃣ 运行完整测试套件

```bash

python test_suite.py

# 方式2: 使用pytest（详细输出）
pytest test_suite.py -v -s

# 方式3: 运行特定测试类
pytest test_suite.py::TestHealthCheck -v -s
pytest test_suite.py::TestAuthEndpoints -v -s
pytest test_suite.py::TestDiagnosisEndpoints -v -s
```

## ⚠️ 故障排除

### 问题1: 连接被拒绝
```
Connection refused at http://localhost:8000
```
**解决方案:**
```bash
# 确保系统已启动
docker ps  # 检查容器状态
./startup.sh  # 重新启动
```

### 问题2: 测试超时
```
Timeout: test did not complete within 5 seconds
```
**解决方案:**
```bash
pytest test_suite.py --timeout=30 -v
```

### 问题3: 认证失败
- 检查数据库连接
- 验证 Redis 服务状态
- 查看 main_app.py 中的认证配置

### 问题4: Agent执行失败
- 检查依赖的 LLM 服务
- 验证 RAG 知识库连接
- 查看 Week 8-12 Agent模块日志

## 💡 最佳实践

✅ **DO:**
- 每次部署前运行完整测试

## 📞 获得帮助

如果测试失败，检查：
1. 系统是否正确启动
2. 所有依赖服务（DB、Redis、LLM）是否可用
3. main_app.py 中的端点配置是否正确
4. Week 1-20 模块依赖是否正确导入
