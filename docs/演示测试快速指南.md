## 📋 五大演示场景

### 场景1️⃣: 轴承故障诊断 ⚠️ WARNING
**真实场景**：工厂生产线主轴承磨损

**关键指标**：
- 振动幅度：0.85mm（严重 > 0.7mm）
- 温度：68°C（警告区间 50-70°C）
- 噪音：82dB（超出正常范围）

**诊断结果**：
- 状态：**WARNING** 
- 根本原因：轴承外圈缺陷
- 置信度：**89%**
- 建议：**48小时内更换**

**测试命令**：
```bash
curl -X POST http://localhost:8000/api/v1/diagnosis/submit \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "BEARING-001",
    "device_name": "生产线主轴承",
    "measurement_data": {
      "vibration_amplitude_mm": 0.85,
      "temperature_celsius": 68,
      "noise_level_db": 82
    }
  }'
```

### 场景2️⃣: 泵机械密封泄漏 🔧 PREDICTIVE_MAINTENANCE
**真实场景**：化工泵密封磨损，需要预防性维护

**关键指标**：
- 密封泄漏速率：2.5ml/h（警告 1-5ml/h）
- 温度：72°C（升高）
- 真空度：-0.2bar（异常）

**成本效益分析**：
- 预防性维修成本：$1,200
- 紧急维修成本：$8,000+
- **节省：$6,800+**

**诊断结果**：
- 状态：**PREDICTIVE_MAINTENANCE**
- 建议：下次维护窗口期间更换密封
- 剩余寿命：**7-14天**

### 场景3️⃣: 电机故障诊断 🔴 CRITICAL
**真实场景**：电机绕组短路 + 轴承磨损

**紧急指标**：
- 相电流不平衡：6.8%（异常 > 5%）
- 总谐波失真(THD)：12.1%（异常 > 5%）
- 绕组温度：85°C（过高，风险：火灾）

**检测到的故障**：
1. ⚠️ 相电流不平衡 (6.8%)
2. ⚠️ 高谐波含量 (12.1% THD)
3. ⚠️ 绕组温度过高 (85°C)
4. ⚠️ 轴承磨损迹象

**紧急行动**：
```
🛑 立即停止运行
⏰ 必须在2小时内停机（防止火灾风险）
🔧 需要紧急维修
```

### 场景4️⃣: 齿轮箱故障诊断 ⚡ URGENT_MAINTENANCE

**关键指标**：

**维修范围**：
- 更换损坏的齿轮
- 进行油液更换
- 检查轴对齐
- 检查并可能更换轴承

**成本估算**：$3,500-5,000
**停机时间**：4-6小时

## 🧪 测试方法对比

### 方法1️⃣: Python脚本直接调用（最快）
```python
from demo_scenarios import BearingFailureDemo

# 获取测试数据
data = BearingFailureDemo.get_demo_data()

# 导入到诊断系统
import requests
response = requests.post(
    "http://localhost:8000/api/v1/diagnosis/submit",
    json=data
)
print(response.json())
```

### 方法2️⃣: curl命令行（最灵活）
```bash
# 测试轴承故障
curl -X POST http://localhost:8000/api/v1/diagnosis/submit \
  -H "Content-Type: application/json" \
  -d @bearing_test.json

# 查询诊断结果
curl -X GET http://localhost:8000/api/v1/diagnosis/diag_001
```

### 方法3️⃣: Swagger UI（最直观）
1. 打开 http://localhost:8000/docs
2. 点击 "Try it out" 按钮
3. 在请求体中粘贴JSON数据
4. 点击 "Execute" 查看结果

## 🤖 Multi-Agent执行链测试

```bash
curl -X POST http://localhost:8000/api/v1/agent/execute \
  -H "Content-Type: application/json" \
  -d '{
    "task": "comprehensive_diagnosis",
    "device_id": "BEARING-001"
  }'
```

**执行顺序**：
1. 数据验证器 → 验证测量数据质量
2. 模式分析器 → 从知识库分析故障模式
3. 风险评估器 → 评估风险等级和紧急程度
4. 行动推荐器 → 推荐维护措施
5. 报告生成器 → 生成综合报告

## 🎯 完整工作流测试

### 步骤1：用户注册登录
```bash
# 注册
curl -X POST http://localhost:8000/api/v1/auth/register \
  -d '{"username":"engineer1","password":"pass123"}'

# 登录获取Token
curl -X POST http://localhost:8000/api/v1/auth/login \
  -d '{"username":"engineer1","password":"pass123"}'
```

### 步骤2：提交诊断请求
